# 적 센싱 비주얼 디버깅 — /prd 입력 답변

> `/prd 적 센싱 비주얼 디버깅` 실행 후 이어지는 질문에 붙여넣을 답변 정리. 근거 설계: 4B-2c6 outline.md(센서는 View 소유·상태/조건은 순수 C#, `EnemyBlackboard`는 센서→AI 한 방향) + 4B-2c5 `enemy-ai-spec.md`(시야·소리·지면/벽 센싱과 순간 신호 vs 보존 값).

| 질문 | 답변 |
|---|---|
| 무엇을 만드는가 | 적의 센싱 상태를 에디터 씬 뷰에서 눈으로 확인하는 디버그 표시. 시야 부채꼴·청취 반경·지면/벽 레이·마지막 인지 위치·현재 상태 이름을 적 위에 그린다. **에디터 씬 뷰에서만 보이며 게임 화면(Game View)·빌드에는 전혀 나오지 않는다** |
| 왜 필요한가 | 센싱 전이는 값이 눈에 보이지 않아 "왜 안 걸렸는지"를 로그로만 쫓게 된다. 시야 각도·거리, 차폐, 청취 반경, 낭떠러지 판정을 씬 뷰에 그려두면 값 조정과 원인 파악을 플레이 중에 바로 한다 |
| 구성요소 | `EnemySensingDebugView`(적 View 오브젝트에 붙는 MonoBehaviour, `OnDrawGizmos`/`OnDrawGizmosSelected`에서만 그림), `SensingDebugSettingsSO`(색·표시 항목 토글·항상 표시 여부), 그리기 데이터 소스 = 해당 적의 `EnemyBlackboard` 읽기 전용 참조 |
| 무엇을 그리는가 | ① 시야 — 바라보는 방향 기준 각도·거리 부채꼴, `IsPlayerVisible`이면 강조색. ② 차폐 — 플레이어까지의 판정 레이와 막힌 지점. ③ 청취 — 청취 반경 원, `HeardSoundThisTick`이면 순간 강조. ④ 지면·벽 — 발밑 앞쪽/전방 레이와 히트 여부(`GroundAhead`·`WallAhead`). ⑤ 어그로·공격 범위 원. ⑥ `LastKnownTargetPosition` 마커와 경과 시간(`TimeSincePerceived`), 현재 상태 이름 라벨 |
| 에디터 전용 보장 | 그리기 코드는 `#if UNITY_EDITOR`로 감싸고 `Gizmos`/`Handles`만 사용한다(런타임 렌더러·LineRenderer·UI 사용 안 함). Gizmo는 씬 뷰에만 그려지고 빌드에는 코드가 포함되지 않는다. 스크립트는 `Editor` 폴더가 아닌 View 옆에 두되 그리기 진입점 전체가 조건부 컴파일 안에 있다 |
| 역할 분리 | 디버그 표시는 **View 계층에만** 존재한다. 상태·조건·`EnemyBlackboard`·`StateMachine`은 디버그를 위해 한 줄도 바뀌지 않는다 — 디버그 View는 Blackboard와 센서 설정값을 **읽기만** 한다. AI 로직이 디버그 표시 여부에 따라 달라지지 않는다 |
| 갱신 주기 | 그리는 값은 적 AI의 센싱 주기에 맞춰 갱신된 Blackboard 값이다. 디버그 View가 자기만 매 프레임 따로 레이를 쏘지 않는다 — 그러면 화면에 보이는 그림과 AI가 실제로 판단한 값이 어긋난다. 씬 뷰 그리기 자체는 매 프레임 일어나되 직전 갱신값을 그대로 그린다 |
| 데이터 | 색·선 굵기·각 항목 표시 토글(시야/청취/지면·벽/범위/라벨)·선택했을 때만 그릴지 항상 그릴지는 `ScriptableObject`로 두고 View가 참조한다. 시야 각도·거리·청취 반경 같은 수치는 새로 정의하지 않고 적 AI가 이미 쓰는 설정값을 그대로 읽는다 |
| 배치 | `Assets/20_ProjectB`의 적 View 옆에 둔다. 공통 프레임워크(`00_CommonFramework`)에는 디버그 표시를 넣지 않는다 |
| 이번 범위 | 씬 뷰 Gizmo 표시까지. 커스텀 에디터 창, 로그 기록·리플레이, 게임 화면 오버레이 디버그 HUD는 범위 밖 |
