# MeleeCombo — /prd 입력 답변

> `/prd 근접 공격 콤보` 실행 후 이어지는 질문에 붙여넣을 답변 정리. 근거 설계: 4B-2c3 대본(Animation Event 판정 + 콤보 입력 구간/버퍼 + HitboxModule·IDamageable 계약 재사용).

| 질문 | 답변 |
|---|---|
| 무엇을 만드는가 | 2D 플랫폼 액션의 플레이어 근접 공격. 기본공격→콤보1→콤보2로 이어지는 3단 근접 콤보 스킬 1종. 데미지 판정 타이밍은 Animation Event가 쥐고, 콤보는 입력 구간과 버퍼로 잇는다 |
| 구성요소 | 근접 콤보 스킬 정의(단계별 데미지·히트박스 크기·콤보 입력 구간, 데이터), 콤보 판단 Module(콤보 단계·입력 구간·버퍼 상태를 쥔 순수 C#), 공격 View(Animator 재생·Animation Event 수신·히트박스 Collider 소유), Project B 전용 발동 경로(공격 입력으로 발동을 거는 어댑터), 재사용 계약: `HitboxModule`(판정 정책)·피격 이벤트(R3)·`IDamageable`(대상 계약) |
| 재사용 vs 전용 경로 | **재사용(공통 계약)**: HitboxModule의 대상·데미지 판정과 중복 타격 방지 정책, 피격 이벤트 단방향 발행, IDamageable 대상 계약. **Project B 전용으로 추가**: ① 입력으로 발동을 거는 경로(공통 `SkillModule`은 쿨다운마다 자동 Activate하므로 그대로 호출하지 않는다) ② 플레이어 View가 소유한 히트박스를 Animation Event로 On/Off 하는 경로(공통 풀 기반 히트박스 View는 만들어질 때 켜져 정해진 수명으로 꺼지므로 명시적 On/Off 경로를 따로 둔다) ③ 콤보 상태 판단 Module. 판정 자체는 전용 경로가 공통 계약을 호출해 처리한다 |
| 판정 타이밍 | 히트박스 On/Off를 코드 타이머로 재지 않고 Animation Event로 처리한다. 각 공격 애니메이션에서 칼날이 지나기 시작하는 프레임에 히트박스 켜기 이벤트, 스윙이 끝나는 프레임에 끄기 이벤트를 단다. 히트박스 활성 구간이 곧 모션의 유효 타격 구간이 되도록 — 애니메이션을 고치면 판정 타이밍도 함께 따라온다 |
| 콤보 타이밍 | 각 공격 애니메이션 후반에 다음 입력을 받는 콤보 입력 구간이 열린다(열림/닫힘도 Animation Event로 표시). 그 구간에 공격 입력이 들어오면 다음 단계로 전이(기본공격→콤보1→콤보2), 구간을 놓치면 대기 자세로 복귀한다. 구간이 열리기 전에 누른 입력은 버퍼로 기록해 뒀다가 열리는 순간 소비한다 — 4B-2c1 점프 버퍼와 같은 계약 |
| 발동 | 쿨다운 자동 발동이 아니라 플레이어의 공격 입력이 발동을 건다. 공통 `SkillModule`의 자동 Tick 발동을 그대로 쓰지 않고, 입력으로 근접 콤보 스킬 1종의 발동을 거는 Project B 전용 경로를 둔다 |
| 판정·피격 | 켜진 히트박스와 겹친 대상 판정, 같은 대상을 한 스윙에 두 번 때리지 않게 막는 정책은 공통 `HitboxModule`을 호출해 처리한다. 피격은 R3로 "대상이 이만큼 맞았다"는 사실만 단방향 발행하고, 체력 감소 등 반응은 구독한 쪽이 처리한다. 피격 대상은 `IDamageable` 계약에만 의존한다 — 공격 로직이 구체 적/타입에 묶이지 않게 |
| 역할 분리 | 콤보 단계·입력 구간·버퍼 판단, 다음 단계 전이 여부는 Unity API에 의존하지 않는 순수 Module이 소유한다(new로 테스트 가능). Animator 재생·Animation Event 수신·히트박스 Collider 소유는 View가 맡는다. 공격 입력은 Reactive 값으로 받아 판단 Module/발동 경로가 구독한다. **Animation Event가 호출하는 함수는 애니메이션을 재생하는 GameObject의 컴포넌트에 있어야 한다** — 그 GameObject에 수신(브리지) 컴포넌트를 두고 순수 콤보 Module로 신호를 전달한다 |
| 데이터 | 콤보 입력 구간 시간·버퍼 시간, 단계별 데미지·히트박스 크기는 코드 상수가 아니라 조절 가능한 값(예: `ScriptableObject`)으로 두고 Context가 입력받아 Module·스킬 정의에 넘긴다 |
| 배치 | 이 근접 콤보는 프로젝트 B에서만 쓰는 로직이므로 코드(콤보 Module·전용 발동 경로·공격 View)는 `Assets/20_ProjectB` 쪽에 둔다. 공통 코어(`00_CommonFramework`)의 `HitboxModule`·피격 이벤트·`IDamageable` 계약은 참조만 한다(재작성하지 않음) |
| 이번 범위 | 근접 콤보 스킬 1종(기본공격→콤보1→콤보2), Animation Event 기반 히트박스 On/Off, 콤보 입력 구간·버퍼까지. 구현 시간에 원거리 투사체 스킬 1종을 함께 얹는데, 이건 공통 `SkillModule`의 쿨다운 발동 경로와 풀 기반 히트박스 View를 그대로 재사용하는 형태다(새 설계 조각 없음). 특수 스킬, 방어·패링, 넉백·경직 연출은 범위 밖(뒤 클립) |
