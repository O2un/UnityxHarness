# ItemDrop — /prd 입력 답변 (PRD 1 · 아이템 뷰 & 데이터)

> `/prd ItemDrop` 실행 후 이어지는 질문에 붙여넣을 답변 정리. 근거 설계: `outline.md`.
> 경험치 루프의 앞단. 출력("경험치 N 획득")이 PRD 2(`ExperienceSystem`)의 입력이 된다.

| 질문 | 답변 |
|---|---|
| 무엇을 만드는가 | 처치 보상 루프의 앞단. 적이 죽은 자리에 경험치 아이템을 스폰하고, 플레이어가 닿으면 주움을 감지해 "경험치 N 획득"을 다음 단계로 넘긴다 |
| 구성요소 | 드롭 아이템 View/Actor(스폰·표시·주움 연출), Context(스폰 트리거·주움 이벤트 수신), 스폰용 `PoolModule` 재사용 |
| 스폰 입력 | 적 Actor가 죽는 위치와 경험치량을 넘겨 `ItemActor`를 스폰. 경험치 양은 몬스터 데이터에서 가져와 주입 |
| 주움 감지 | 아이템의 View/Context가 `OnTriggerEnter`로 플레이어 접촉을 감지하고, 아이템 Actor가 주움 이벤트를 발행 |
| 데이터 | 경험치 양은 몬스터 데이터에서 스폰 시 주입. 아이템 프리팹·풀 크기 등 스폰 파라미터도 데이터로 구성 |
| 배치 | 드롭 스폰·주움 감지·표시는 View/Context. 스폰은 앞서 만든 `PoolModule`을 재사용 |
| 입력(Depends On) | 적 처치 — 죽는 위치·경험치량 |
| 출력(Provides) | "경험치 N 획득"(amount) 입력 하나 → PRD 2 |
| 금지 | 경험치 누적·레벨 판정 로직을 여기서 하지 않음. `ExperienceModule` 내부 참조 금지 |
