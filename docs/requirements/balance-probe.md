# balance-probe PRD

> 실제 프로젝트 기준 위치: `docs/requirements/balance-probe.md`

## Overview

프로젝트 A를 사람이 직접 플레이하는 동안, 실제 조작이 섞인 밸런스 실측 데이터(구간별 처치·데미지·동시 생존 등)를 코드가 자동으로 기록하고 파일로 내보내는 계측 시스템이다. 내보낸 파일을 분석해 `wave-balance-prep.md`의 계산 기댓값(pressure·기대 DPS·예상 peak alive)과 실제 플레이가 얼마나 맞는지 대조한다.

**이 기능은 밸런스 튜닝을 위한 에디터 전용 계측 도구다.** 게임 빌드에 포함되는 런타임 기능이 아니므로, 이 프로젝트의 아키텍처 규칙(Service 인터페이스 추상화, DI 강제, 순수 Module 강제 등)을 **따르지 않아도 된다.** 최대한 간소하게 만든다. (4A-4c4의 에디터용 리소스 브라우저도 동일한 규칙 예외를 적용한다.)

처리 순서:
1. **사람이 직접 플레이** — 한 판을 시작→클리어/게임오버까지 실제로 조작.
2. **코드가 실측 기록** — 기존 게임 이벤트를 구독해 구간별 지표를 집계.
3. **데이터 분석** — 판 종료 시 CSV/JSON으로 내보내고, 계산 기댓값과 대조.

## Goals

- 한 판을 플레이하면 웨이브 구간별 실측 지표가 자동으로 수집된다.
- 판 종료(클리어/게임오버) 시 결과가 CSV 또는 JSON 파일로 내보내진다.
- 수집은 기존 게임 로직을 건드리지 않고 **이벤트 구독만으로** 이뤄진다.
- 내보낸 파일이 `wave-balance-prep.md`의 계산 기댓값과 같은 축(웨이브 구간·지표)으로 대조 가능하다.

## Out of Scope

- 계산 기댓값 산출 자체 (그건 `/balance-eval`이 `wave-balance-prep.md`로 수행 — 이 계측과 별개)
- 밸런스 수치의 자동 조정·튜닝 (측정만 하고 값은 바꾸지 않는다)
- 게임 빌드 포함·런타임 상시 동작 (에디터 플레이 모드 전용)
- 실시간 그래프·대시보드 UI (파일 출력까지만)
- 파일 저장을 위한 SaveService/파일 IO 추상화 계층 (에디터 전용이므로 직접 파일 쓰기 허용)

## Technical Requirements

- **배치**: `Assets/10_ProjectA/` 전용. 공통 프레임워크에 넣지 않는다.
  - `BalanceProbeManager` — 수집 담당. 게임 이벤트를 구독하고 현재 시각(플레이 경과 시간)으로 웨이브 구간을 판별해 표본을 모은다.
  - `BalanceProbeModule` (또는 동급의 집계 로직) — 모은 표본을 구간별 지표로 집계. (에디터 전용이라 순수 Module 분리는 강제 아님 — 간소하면 Manager에 직접 둬도 됨.)
- **데이터 소스 (기존 이벤트 구독, 게임 로직 비침범)**:
  - 처치: `IEnemyKillEvent.OnKilled` (`EnemyKilledInfo`: position, exp)
  - 레벨·레벨업: `ExperienceModule.CurrentLevel`, `OnLevelUp`
  - 데미지(=플레이어 DPS 산출 소스): 적 `IHealth.VaryHP`/`CurrentHP` 변화. 플레이어가 적에게 준 데미지 총량을 시간으로 나눠 실측 DPS를 낸다.
  - 동시 생존 수(peak alive): `EnemySpawnManager`의 활성 적 수. 구간별 최고치를 기록.
- **웨이브 구간 분할**: `WaveData_Test` 기준 5개 구간(1–12 / 14–26 / 28–40 / 42–54 / 56–70초). 표본을 이 구간에 매핑해 구간별로 지표를 나눈다.
- **기록 지표(구간별)**: 실측 DPS, 처치 수, 동시 타격 수(가능하면), peak alive, 그 구간의 플레이어 레벨·업그레이드 상태, 생존/피격 여부.
- **출력**: 판 종료 시 CSV 또는 JSON 1개 파일. 경로·파일명은 에디터에서 확인 가능한 위치(예: 프로젝트 루트나 지정 폴더). 컬럼은 웨이브 구간을 행으로, 지표를 열로.
- **동작 트리거**: 에디터 플레이 모드에서만. 판 시작 시 수집 시작, 종료 이벤트에서 파일 출력.

```
# 출력 예 (CSV)
wave,seconds,measuredDps,kills,peakAlive,level
1,1-12,,,,
2,14-26,,,,
...
```

## Acceptance Criteria

- [ ] 에디터에서 한 판을 플레이하면 5개 웨이브 구간별 표본이 수집된다.
- [ ] 판 종료 시 구간별 지표가 담긴 CSV(또는 JSON) 파일이 생성된다.
- [ ] 수집이 기존 게임 클래스의 로직을 수정하지 않고 이벤트 구독만으로 이뤄진다(비침범).
- [ ] 코드가 `Assets/10_ProjectA/` 안에 있고 공통 프레임워크를 오염시키지 않는다.
- [ ] 내보낸 파일의 지표 축(웨이브 구간·DPS·peak alive)이 `wave-balance-prep.md`와 대조 가능하다.
- [ ] 게임 빌드/런타임 상시 동작에 영향을 주지 않는다(에디터 플레이 모드 전용).

## Open Questions

- 동시 타격 수는 스킬 히트박스 구조상 이벤트로 바로 잡히지 않을 수 있다 — 잡기 어려우면 이 지표만 `Needs Assumption`으로 남기고 다른 지표부터 채운다.
- 출력 파일 경로를 프로젝트 루트로 할지 지정 폴더(예: `BalanceLogs/`)로 할지 — 구현 직전 결정.
- 판 종료 이벤트(클리어·게임오버) 훅 지점을 `GameManager`/`EnemySpawnManager.OnCleared` 중 어디로 걸지 — 구현 시 확인.
