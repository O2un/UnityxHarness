# Project A 웨이브 밸런스 Eval 결과

평가 대상: `WaveData_Test`, `MonsterData_*`, 플레이어 스킬 3종(`MeleeSwingSkill`/`ProjectileSkill`/`AuraFieldSkill`) + 업그레이드 SO. 모든 수치는 애셋에서 직접 읽음(추측 없음). 동적 플레이 실측은 범위 밖.

## 검증한 입력 애셋 값

몬스터 HP (`_maxHp`): Slime 30, Dash 15, Armored 60 — 프렙 §1과 일치.
웨이브 구성/도착HP: 프렙 §2 표와 일치. **단, 구간 길이는 프렙이 "대개 12초"라 했으나 실제 W1=1–12(11초), W5=56–70(14초)** 로 다름.

플레이어 스킬 base:
- Melee: dmg 10 / cd 1.0 / range 2 → DPS 10.0
- Projectile: dmg 8 / cd 0.8 → DPS 10.0
- Aura: dmg 5 / reHit 0.5 / **cooldown 3 / lifetime 2** → raw 10/s

스킬 발동 로직(`SkillModule.Tick`): `Timer >= Cooldown`마다 `Activate` 재호출. Aura는 lifetime 2s 히트박스를 cd 3s마다 재생성 → **duty = lifetime/cooldown = 2/3**. 지속 Aura 실효 DPS = (dmg/reHit)×(2/3) = 10×0.667 = **6.67/s**.

업그레이드 적용(`SkillStats.ApplyUpgrade`): 레벨당 델타가 int/float로 **누적** 적용되며, `upgrade.Level <= Level`이면 무시. `LevelUpSkillPool`은 레벨업 시 후보 중 **한 경로만** 선택 → Lv2는 단일 선택 델타 하나만 적용됨.

## 판정 항목

### [Issue-1] 프렙 문서의 Lv2 누적 DPS가 단일 업그레이드 경로로 도달 불가 (기댓값 과대 ~2배)
- **대조**: 플레이어 DPS 산정 근거 전체(전 웨이브에 영향).
- **기댓값 vs 애셋**: 프렙 §3은 Melee Lv2 "26/0.6→43", Projectile Lv2 "23/0.3→77"로 기재. 그러나 `SkillModule`+`LevelUpSkillPool`상 레벨업은 후보 중 **한 경로만** 적용(`ApplyUpgrade`는 델타 1개 누적).
  - Melee Lv2 단일 경로 최대: Heavy(+10dmg) = 20/1.0 → **20.0** (Swing 18.8 / Rapid 13.3). 43 도달 불가.
  - Projectile Lv2 단일 경로 최대: Impact(+10dmg) = 18/0.8 → **22.5** (Lv2 21.7 / Rapid 16.0). 77 도달 불가.
- **영향**: 프렙 §3의 Lv2 DPS로 pressure를 낮게 산출하면 실제보다 난이도를 과소평가. pressure 재계산은 위 단일-경로 DPS로 해야 함.
- **가정**: 레벨당 1선택. 만약 설계 의도가 "레벨당 3개 전부 적용"이면 코드(`LevelUpSkillPool`/선택 UI)와 프렙이 어긋난 것.

### [Issue-2] Aura DPS 표기 오류 (cooldown/lifetime duty-cycle 미반영)
- **대조**: 플레이어 지속 DPS 근거(전 웨이브).
- **기댓값 vs 애셋**: 프렙 §3은 Aura를 "5 / reHit 0.5 → 10/s (범위 유지 가정)"로 연속 10/s 취급. 실제 애셋은 `cooldown 3`, `lifetime 2`. `SkillModule.Tick`이 cd 3s마다 재생성하므로 히트박스는 3s 중 2s만 존재 → **duty 2/3, 실효 6.67/s**. 프렙의 지속 DPS 공식 "Damage/reHit"에 duty 항이 빠짐.
- **영향**: 3종 콤보 기대 DPS가 30이 아니라 26.67. 콤보 pressure가 프렙 기준보다 상향(더 위험).
- **가정**: lifetime<cooldown 관계가 의도라면 duty 항을 공식에 넣어야 함. lifetime≥cooldown으로 연속 유지가 의도였다면 애셋(lifetime 2 또는 cooldown 3)이 잘못됨.

### [Issue-3] 초반 웨이브가 의도한 "안정" 구간을 못 만듦 (단일 스킬 보유 시 전 구간 위험)
- **대조**: W1~W2 (초반).
- **기댓값 vs 임계값**: 신규 플레이어가 스킬 1종만 든 상태의 Lv1 DPS=10. pressure W1=2.18, W2=2.00 (k=1). k=0.6이면 3.3~3.6. 모두 위험(>1.2) 임계 크게 초과.
- **근거**: 초당 유입 W1 21.82 / W2 20.00 vs 단일 DPS 10.
- **판정**: 초반이 "안정(<0.8)"으로 시작해 후반 상승하는 흐름과 어긋남. 3종을 모두 확보해야만 W1~W2가 경계(0.73~0.82)로 내려오므로, **초기 스킬 지급 수**가 밸런스 전제. 프렙에 초기 보유 스킬 수 가정이 없음.
- **가정**: 게임 시작 시 몇 개 스킬을 들고 시작하는지(1종 vs 3종). 이 값이 결과를 좌우 → 확인 필요.

### [Issue-4] 중·후반 pressure가 콤보 최대치에서도 임계 초과 (난이도 상승이 아니라 처치 불능 방향)
- **대조**: W3~W5 (중반~후반).
- **기댓값 vs 임계값**: 3종 콤보 duty 반영, k=1.0(범위·명중 완벽): W3 1.22 / W4 1.64 / W5 2.05 — 모두 위험(>1.2). k=0.6: W3 2.03 / W4 2.73 / W5 3.41.
- **근거**: 유입 32.5/43.75/54.64 vs 콤보 실효 26.67. Lv2 단일 경로로 최상 조합(Heavy20+Impact22.5+AuraPower10.67≈53.2)을 가정해도 W5 54.64는 여전히 p≈1.03(경계 상단), k<1이면 위험.
- **판정**: pressure가 초반→후반으로 "상승"은 하나(0.82→2.05), 의도한 밴드(경계 유지)를 벗어나 후반이 처치 속도 부족. 성장(레벨업) 속도가 유입 증가를 못 따라가는지가 관건 — 동시 타격 수 `n`과 레벨 도달 시점에 크게 의존.

### [Needs Assumption-1] 동시 타격 수(n) 미확정 — pressure 밴드가 n에 지배됨
- **대조**: 전 웨이브.
- **내용**: 프렙 pressure 공식은 `기대DPS × 동시 타격 수`. Melee 히트박스(range 2)와 Aura는 광역이라 범위 내 다수 적을 동시 타격 가능하나, 애셋에는 최대 동시 타격 수 상한이 없음. n이 곧 pressure 분모의 배수.
  - 예: W5 유입 54.64, 콤보 26.67, n=2·k=1이면 p=1.02(경계), n=3이면 p=0.68(안정).
- **판정**: 밴드로만 판단 가능. Melee/Aura의 실제 평균 동시 타격 수(적 밀집도·히트박스 크기)는 조작·배치 의존 → **동적 플레이 실측 필요, 범위 밖(별도 작업)**.
- **가정**: 위 pressure 표는 n=1(가장 보수적). n≥2면 후반도 경계로 내려올 여지 있음. 이 가정이 W3~W5 판정(Issue-4)을 뒤집을 수 있으므로 실측 우선순위 최상.

### [Needs Assumption-2] expectedPeakAlive / pool prewarm — 처치율·레벨 타이밍 불확정으로 밴드만 제시
- **대조**: 전 웨이브, poolKey 3종.
- **계산**: NormalSpread(`WaveModule.ResolveSpawnTime`)로 각 웨이브가 구간에 정규분포(mean=중앙, σ=구간/6) 분산 스폰. 처치율 = 종별 DPS배분 / 평균HP. 상한(처치 0 가정, 최대 동시 생존)은 웨이브 총 마리수:
  - `Enemy_Slime`: 단일 웨이브 최대 Count 10(W5) → 처치 반영 전 상한 10.
  - `Enemy_Dash`: 최대 7(W5) → 상한 7.
  - `Enemy_ArmoredMelee`: 최대 6(W5) → 상한 6.
  - 역산 prewarm(×1.2, ceil): Slime ceil(10×1.2)=**12**, Dash ceil(7×1.2)=**9**, Armored ceil(6×1.2)=**8**. (처치를 전혀 반영 안 한 안전 상한)
  - 처치 반영 시(콤보 DPS로 Armored 60HP 처치에 ~2.3s 소요, Slime 30HP ~1.1s): expectedPeakAlive는 이보다 작아지나 Issue-4대로 후반 pressure>1이면 실제 생존 수가 유입 쪽으로 누적되어 상한(12/9/8)에 근접.
- **판정**: 처치율이 Issue-1·2·4(DPS 불확정)와 동시 타격 수 n에 의존하므로 단일값 불가. **prewarm은 안전 상한 12/9/8을 기준**으로 두되, 실제 peakAlive는 동적 실측으로 확정 필요(범위 밖).
- **가정**: 여유계수 1.2(프렙 기본), NormalSpread로 순간 폭증 없음(σ=구간/6이라 mean±3σ가 구간 경계). 순간 동시 스폰 상한은 이 분포로 완화됨.

## 범위 밖(별도 작업으로 분리)
- 동시 타격 수 n의 실측(Melee range 2 / Aura 범위의 평균 적중 적 수) — Needs Assumption-1·2 판정을 좌우.
- 레벨업 도달 시점(경험치 곡선 vs 웨이브 진행) 실측 — 후반 DPS 성장이 유입을 따라가는지 확정.
- 명중·범위 유지율 k의 실제 분포 — 위 밴드를 단일값으로 좁히려면 필요.

## 프렙 문서 자체의 데이터 정합성 지적(수치 변경 없이 기록만)
1. 구간 길이: W1=11s, W5=14s로 "대개 12초" 서술과 불일치 → inflow 계산 시 구간별 실제 길이 사용해야 함(본 eval은 실제값 적용).
2. §3 Lv2 DPS(Melee 43, Proj 77)는 단일 업그레이드 경로로 도달 불가(Issue-1).
3. §3 Aura "10/s"는 duty-cycle 미반영(Issue-2).
