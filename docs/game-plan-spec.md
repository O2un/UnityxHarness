# /game-plan 스킬 명세

skill-creator에게 넘길 명세 파일입니다. 이 파일 자체가 "무엇을 · 어떻게 · 제약" 구조의 기획 문서 형식입니다.

## 무엇을 (스킬 정의)

- **이름**: `game-plan`
- **호출 형태**: `/game-plan`
- **트리거(description)**: 게임 루프·기획 문서 작성을 요청할 때 호출
- **목적**: Unity 게임의 핵심 루프와 기능 우선순위를 정리한 기획 문서를 `docs/design/game-plan.md`로 생성한다.

## 입력

사용자가 다음 세 가지를 입력으로 준다.

- 장르
- 핵심 루프
- 주요 시스템 목록

## 어떻게 (동작 절차)

1. 입력받은 장르와 핵심 루프를 파악한다.
2. 같은 장르에 여러 대표작이 있을 수 있으므로, 인터넷에서 유사 장르 게임을 찾아 레퍼런스 후보를 3~5개 제시한다. 각 후보의 특징(핵심 루프·차별점)을 한 줄로 붙인다.
3. 사용자에게 **어떤 게임을 레퍼런스로 삼을지 다시 질문**해 선정받는다. 선정된 게임의 루프·시스템 구성을 이후 기획에 반영한다.
4. 핵심 루프를 단계별 순서도(텍스트)로 정리한다.
5. 주요 시스템을 각 시스템의 역할과 함께 정리한다.
6. 기능을 **MVP**와 **나중**으로 나눈다.
7. 구현할 **개발 순서**를 번호로 제안한다.
8. 초안을 사용자에게 보여주고 검토를 요청한 뒤, 승인되면 파일로 저장한다.

## 출력 문서 구조 (`docs/design/game-plan.md`)

- **Game Concept**: 장르의 핵심 경험을 한 단락으로.
- **Reference Games**: 사용자가 선정한 레퍼런스 게임과 각 게임에서 참고할 요소.
- **Core Loop**: 각 단계에서 무엇이 일어나는지 순서도로.
- **Key Systems**: 시스템별 역할 (구현 클립의 단위가 됨).
- **Feature Priority**: MVP / 나중 2분할. 이 문서에서 가장 중요한 섹션.
- **Development Order**: 구현 순서를 번호로.

## 제약

- 레퍼런스 게임은 임의로 확정하지 말고, 인터넷 검색으로 후보를 제시한 뒤 사용자에게 선정받는다.
- 문서는 반드시 `docs/design/`에 저장한다. 폴더가 없으면 자동 생성한다.
- 문서는 한국어로 작성한다.
- MVP 기준은 사용자와 합의한다 — 임의로 범위를 확정하지 않는다.
- 알 수 없는 값은 추측하지 말고 질문한다.
