# Task Breakdown — 3D 뱀파이어 서바이버류 MVP

> [game-plan.md](./game-plan.md)의 **Development Order**를 따르는 구현 태스크 분해 문서.
> MVP 6개 시스템 전체를 포함하며, 각 태스크의 **순서 · 시스템 · 내용 · 컨텐츠 의존성**을 명시한다.
> 코드 배치 위치(`00_CommonFramework` vs `10_ProjectA`)는 각 시스템 구현 직전 사용자에게 확인한다. (하네스 오픈 퀘스천)

---

## 용어

- **컨텐츠 의존성**: 해당 시스템이 동작·검증되기 위해 먼저 존재해야 하는 다른 시스템·데이터·씬 요소.
- 각 태스크는 `Manager / Module / Service` 3레이어 분리를 전제로 하되, 기능이 단순하면 Module 분리를 생략할 수 있다. (game-plan / CLAUDE.md 규칙)

---

## 의존성 그래프 (요약)

```
①플레이어 이동
   ↓
②적 스폰 + ③추격 AI  (①의 플레이어 위치 참조)
   ↓
④체력 & 피격  (②③의 적, ①의 플레이어)
   ↓
⑤자동 공격  (④의 피격 처리에 얹힘)
   ↓
⑥경험치 & 레벨업  (④사망·⑤처치 이벤트 소비)
   ↓
⑦게임오버/결과 화면  (④플레이어 사망 이벤트 소비)
```

---

## Task 1 — 플레이어 이동 (Topdown3D)

| 항목 | 내용 |
|---|---|
| 순서 | 1 (최우선 · 나머지 모든 것의 기반) |
| 시스템 | 플레이어 이동 |
| 목표 | 입력을 받아 캐릭터를 탑다운 3D 평면(XZ)에서 이동시킨다 |

**내용**
- `InputReader`(Service, 인터페이스 `IInputReader`): 이동 입력 벡터 제공
- `PlayerMover`(Manager, MonoBehaviour 소유): 입력 벡터 → XZ 평면 이동, 이동 방향으로 회전
- 이동 로직이 단순하면 Module 생략, 속도/가속 등이 붙으면 `MovementModule` 분리
- 카메라: 플레이어를 따라가는 탑다운 고정 앵글

**컨텐츠 의존성**
- 없음 (최초 태스크)
- 씬: 바닥 평면, 플레이어 GameObject, 탑다운 카메라, Directional Light

**검증 게이트**: 컴파일 → Play → WASD로 플레이어가 평면을 이동

## Task 2 — 적 스폰 & 웨이브

| 항목 | 내용 |
|---|---|
| 순서 | 2 (상대할 대상 확보) |
| 시스템 | 적 스폰 |
| 목표 | 시간에 따라 적을 화면 밖에서 생성한다 (MVP: 단순 연속 스폰, 최소 스케일링) |

**내용**
- `EnemySpawner`(Manager): 일정 간격으로 플레이어 주변 원주에 적 생성
- `SpawnModule`(Module): 스폰 위치 계산, 시간에 따른 스폰 간격/수 스케일링(최소)
- 적 프리팹 참조는 `AssetService`(Addressables) 통해 로드 (`Resources.Load` 금지)
- 오브젝트 풀링은 MVP 범위에서 선택 (다수 적 성능 이슈 시 도입)

**컨텐츠 의존성**
- **Task 1**: 스폰 기준 위치로 플레이어 Transform 필요
- 에셋: 적 프리팹(플레이스홀더 캡슐/구体 가능)

**검증 게이트**: Play 시 적이 주기적으로 생성되어 씬에 나타남

---

## Task 3 — 적 추격 AI

| 항목 | 내용 |
|---|---|
| 순서 | 2와 한 묶음 (스폰된 적이 의미를 갖게) |
| 시스템 | 적 AI (추격) |
| 목표 | 스폰된 적이 플레이어를 향해 이동한다 |

**내용**
- `EnemyAgent`(Manager, MonoBehaviour 소유): 매 프레임 플레이어 방향으로 이동
- `ChaseModule`(Module): 방향·속도 계산 (순수 로직)
- 플레이어 위치는 DI 주입된 참조로 획득 (`FindObjectOfType` 금지)

**컨텐츠 의존성**
- **Task 1**: 추격 목표인 플레이어 위치
- **Task 2**: 추격할 적 인스턴스

**검증 게이트**: 스폰된 적이 플레이어를 향해 몰려옴

---

## Task 4 — 체력 & 피격

| 항목 | 내용 |
|---|---|
| 순서 | 3 (플레이어/적이 죽을 수 있게) |
| 시스템 | 체력 & 피격 |
| 목표 | 플레이어·적의 체력 관리, 피격·사망 처리. 플레이어 사망 시 게임오버 트리거 |

**내용**
- `HealthModule`(Module, 순수 로직): 현재/최대 체력, `TakeDamage`, 사망 판정
- `Health`(Manager, MonoBehaviour 소유): HealthModule 위임, 사망 이벤트 발행(R3 `Subject`/`ReactiveProperty`)
- 적 → 플레이어 접촉 피해 처리 (충돌 or 근접 판정)
- 사망 이벤트: 적 사망 → 처치 보상 트리거용 / 플레이어 사망 → 게임오버용

**컨텐츠 의존성**
- **Task 1**: 피해를 입는 플레이어
- **Task 2/3**: 플레이어에게 접촉해 피해를 주는 적
- 이 태스크가 **자동 공격(Task 5)과 경험치(Task 6)의 전제**

**검증 게이트**: 적 접촉 시 플레이어 체력 감소, 체력 0 시 사망 이벤트 발행

---

## Task 5 — 자동 공격

| 항목 | 내용 |
|---|---|
| 순서 | 4 (피격 시스템 위에 얹힘 — 적을 처치 가능하게) |
| 시스템 | 자동 공격 |
| 목표 | 사거리 내 가장 가까운 적을 자동 탐색·공격한다 (MVP: 단일 무기) |

**내용**
- `AutoAttack`(Manager): 쿨다운마다 사거리 내 대상 탐색 → 공격 실행
- `TargetingModule`(Module): 가장 가까운 적 선정 (순수 계산)
- 공격 결과는 대상의 `Health.TakeDamage` 호출로 연결 (Task 4 재사용)
- 투사체/즉시타격 중 MVP는 즉시타격 or 단순 투사체 택1

**컨텐츠 의존성**
- **Task 4**: 데미지를 적용할 체력·피격 시스템
- **Task 2/3**: 공격 대상인 적
- **Task 1**: 공격 원점인 플레이어 위치

**검증 게이트**: 사거리 내 적이 자동으로 피해를 입고 처치됨

---

## Task 6 — 경험치 & 레벨업

| 항목 | 내용 |
|---|---|
| 순서 | 5 (처치에 보상·성장 부여, 루프 완성) |
| 시스템 | 경험치 & 레벨업 |
| 목표 | 처치 보상(경험치) 수집, 레벨업 트리거, 레벨업 시 능력 1~3택 제공 |

**내용**
- 적 사망 시 경험치 드롭 생성 → 플레이어 근접 시 수집
- `ExpModule`(Module, 순수 로직): 누적 경험치, 레벨업 임계값, 레벨업 판정
- `LevelManager`(Manager): 경험치 수집 이벤트 소비, 레벨업 시 능력 선택 UI 트리거
- 능력 선택: MVP는 1~3택 (예: 공격력↑, 이동속도↑, 체력↑) 적용
- 선택 결과가 Task 1/4/5의 수치에 반영되도록 연결

**컨텐츠 의존성**
- **Task 4**: 적 사망 이벤트 (경험치 드롭 발생원)
- **Task 5**: 적을 실제로 처치하는 수단
- **Task 1**: 드롭 수집을 위한 플레이어 이동
- 에셋/UI: 경험치 드롭 오브젝트, 레벨업 선택 UI

**검증 게이트**: 처치 → 경험치 획득 → 레벨업 시 선택지 표시 → 선택 능력 반영

---

## Task 7 — 게임오버 / 결과 화면

| 항목 | 내용 |
|---|---|
| 순서 | 6 (한 판 마무리) |
| 시스템 | 게임오버/결과 화면 |
| 목표 | 플레이어 사망 시 게임을 정지하고 결과 화면을 표시한다 |

**내용**
- `GameManager`(전역 싱글턴 Manager, VContainer 등록): 게임 상태(진행/종료) 관리
- `GameFlowModule`(Module): 상태 전이 로직
- 플레이어 사망 이벤트 구독 → 게임 정지 → 결과 화면(생존 시간·레벨 등 표시)
- 재시작 버튼(선택)

**컨텐츠 의존성**
- **Task 4**: 플레이어 사망 이벤트
- **Task 6**: 결과 화면에 표시할 생존 지표(레벨 등, 선택)
- UI: 게임오버 화면

**검증 게이트**: 플레이어 사망 → 게임 정지 → 결과 화면 표시
