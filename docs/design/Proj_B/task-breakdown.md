# Task Breakdown — 2D 횡스크롤 플랫폼 액션 로그라이크 MVP

> [game-plan.md](./game-plan.md)의 **Development Order**를 따르는 구현 태스크 분해 문서.
> MVP 6개 시스템 전체를 포함하며, 각 태스크의 **순서 · 시스템 · 내용 · 컨텐츠 의존성**을 명시한다.
> 코드 배치 위치(`00_CommonFramework` vs `10_ProjectA`)는 각 시스템 구현 직전 사용자에게 확인한다. (하네스 오픈 퀘스천)

---

## 용어

- **컨텐츠 의존성**: 해당 시스템이 동작·검증되기 위해 먼저 존재해야 하는 다른 시스템·데이터·씬 요소.
- 각 태스크는 `Manager / Module / Service` 3레이어 분리를 전제로 하되, 기능이 단순하면 Module 분리를 생략할 수 있다. (game-plan / CLAUDE.md 규칙)

---

## 의존성 그래프 (요약)

```
①2D 이동 + 점프 (땅 감지)
   ↓
②체력 & 피격 & 사망  (①의 플레이어)
   ↓
③근접 콤보 공격  (②의 피격 처리에 얹힘)
   ↓
④적 AI (추격 + 근접)  (①의 플레이어 위치, ②의 체력, ③으로 처치)
   ↓
⑤스킬 1종 (쿨다운)  (②③④ 위에 전투 수단 확장)
   ↓
⑥스테이지 클리어 판정 (적 전멸)  (④의 적 사망 이벤트 소비)
   ↓
⑦보상 선택 (2~3택)  (⑥클리어 이벤트 소비, ①③⑤ 수치에 반영)
```

## Task 1 — 2D 이동 + 점프 (땅 감지)

| 항목 | 내용 |
|---|---|
| 순서 | 1 (최우선 · 나머지 모든 것의 기반) |
| 시스템 | 플레이어 이동·점프 (2D Rigidbody) |
| 목표 | 입력을 받아 2D 물리로 좌우 이동·점프시키고, 땅 감지로 점프 가능 여부를 판정한다 |

**내용**
- `InputReader`(Service, 인터페이스 `IInputReader`): 이동 입력(좌우 축) · 점프 입력 제공
- `PlayerMover`(Manager, MonoBehaviour 소유): 입력 → `Rigidbody2D` 좌우 속도 제어 · 점프 임펄스
- `GroundCheck`(Module 또는 MonoBehaviour 협력): 땅 감지(Raycast/OverlapCircle)로 착지 여부 판정 → 점프 가능 조건
- 이동 로직이 단순하면 Module 생략, 가속/코요테 타임/가변 점프 등이 붙으면 `MovementModule` 분리
- 카메라: 플레이어를 따라가는 2D 횡스크롤 추적

**컨텐츠 의존성**
- 없음 (최초 태스크)
- 씬: 바닥/플랫폼 콜라이더, 플레이어 GameObject(`Rigidbody2D`+`Collider2D`), 2D 카메라

**검증 게이트**: 컴파일 → Play → 좌우 이동 + 땅 위에서만 점프

---

## Task 2 — 체력 & 피격 & 사망

| 항목 | 내용 |
|---|---|
| 순서 | 2 (플레이어/적이 죽을 수 있게 — 전투·적 AI의 전제) |
| 시스템 | 체력 & 피격 & 사망 |
| 목표 | 플레이어·적의 체력 관리, 피격·사망 처리. 사망 시 이벤트 발행 |

**내용**
- `HealthModule`(Module, 순수 로직): 현재/최대 체력, `TakeDamage`, 사망 판정
- `Health`(Manager, MonoBehaviour 소유): HealthModule 위임, 사망 이벤트 발행(R3 `Subject`/`ReactiveProperty`)
- 피격 처리: 데미지 적용, 무적 시간(선택), 넉백(선택)
- 사망 이벤트: 적 사망 → 클리어 판정용 / 플레이어 사망 → 재시작(로그라이크)용

**컨텐츠 의존성**
- **Task 1**: 피해를 입는 플레이어
- 이 태스크가 **근접 콤보(Task 3) · 적 AI(Task 4) · 클리어 판정(Task 6)의 전제**

**검증 게이트**: 임의 데미지 적용 시 체력 감소, 체력 0 시 사망 이벤트 발행

---

## Task 3 — 근접 콤보 공격

| 항목 | 내용 |
|---|---|
| 순서 | 3 (피격 시스템 위에 얹힘 — 적을 처치 가능하게) |
| 시스템 | 근접 콤보 공격 |
| 목표 | 공격 입력을 받아 2~3타 콤보를 순서대로 발동하고 판정 범위 내 적에게 피해를 준다 |

**내용**
- `PlayerAttack`(Manager, MonoBehaviour 소유): 공격 입력 수신 → 콤보 진행 → 판정 실행
- `ComboModule`(Module, 순수 로직): 콤보 단계 상태(현재 타수), 입력 윈도우/타이밍, 다음 타 진행·리셋 판정
- 근접 판정: `OverlapBox`/`OverlapCircle` 등 2D 히트박스로 대상 탐색 → `Health.TakeDamage` 호출 (Task 2 재사용)
- 애니메이션 연동은 플레이스홀더 수준으로 최소화

**컨텐츠 의존성**
- **Task 1**: 공격 원점·방향인 플레이어
- **Task 2**: 데미지를 적용할 체력·피격 시스템
- **Task 4**(검증 시): 실제로 때릴 대상인 적 — 그 전엔 더미 대상으로 검증

**검증 게이트**: 공격 입력 시 2~3타 콤보 발동, 범위 내 대상 체력 감소

---

## Task 4 — 적 AI (추격 + 근접)

| 항목 | 내용 |
|---|---|
| 순서 | 4 (상대할 대상 확보 — 피격을 받아 죽는 적) |
| 시스템 | 적 AI (추격 + 근접) |
| 목표 | 적이 플레이어를 추격하고 근접 범위에서 공격한다. 피격을 받아 사망한다 |

**내용**
- `EnemyAgent`(Manager, MonoBehaviour 소유): 상태(추격/공격) 갱신, 이동·공격 실행
- `ChaseModule`(Module, 순수 로직): 플레이어 방향·속도 계산 (2D)
- 근접 공격: 사거리 진입 시 플레이어 `Health.TakeDamage` 호출
- 적도 `Health`(Task 2) 보유 → 플레이어 콤보(Task 3)에 맞아 사망
- 플레이어 위치는 DI 주입된 참조로 획득 (`FindObjectOfType` 금지)
- 적 프리팹 참조는 `AssetService`(Addressables) 통해 로드 (`Resources.Load` 금지)

**컨텐츠 의존성**
- **Task 1**: 추격 목표인 플레이어 위치
- **Task 2**: 적의 체력·사망, 플레이어에게 주는 피해
- **Task 3**: 적을 처치하는 수단
- 에셋: 적 프리팹(플레이스홀더 가능)

**검증 게이트**: 적이 플레이어를 추격 → 근접 시 플레이어 피해 / 콤보로 적 처치

---

## Task 5 — 스킬 1종 (쿨다운)

| 항목 | 내용 |
|---|---|
| 순서 | 5 (전투 수단 확장) |
| 시스템 | 스킬 (쿨다운) |
| 목표 | 스킬 입력 시 능력을 발동하고 쿨다운을 관리한다 (MVP 1종) |

**내용**
- `SkillManager`(Manager, MonoBehaviour 소유): 스킬 입력 수신 → 쿨다운 확인 → 발동
- `CooldownModule`(Module, 순수 로직): 쿨다운 경과·발동 가능 여부 판정
- MVP 스킬 1종 효과: 범위 피해 or 돌진 등 단순 형태 택1 → 대상 `Health.TakeDamage`(Task 2) 재사용
- 쿨다운 상태는 R3 `ReactiveProperty`로 노출해 UI 표현(선택) 대비
- 스킬 슬롯은 1개만 구현 (2종째는 Later)

**컨텐츠 의존성**
- **Task 1**: 발동 원점인 플레이어
- **Task 2**: 데미지 적용 대상 체력 시스템
- **Task 4**: 스킬을 맞을 적

**검증 게이트**: 스킬 입력 시 발동 + 쿨다운 동안 재발동 차단, 적에게 효과 적용

---

## Task 6 — 스테이지 클리어 판정 (적 전멸)

| 항목 | 내용 |
|---|---|
| 순서 | 6 (한 판의 끝을 정의) |
| 시스템 | 스테이지 클리어 판정 |
| 목표 | 스테이지 내 적이 전멸했는지 판정해 클리어를 트리거한다 |

**내용**
- `StageManager`(전역/씬 스코프 Manager, VContainer 등록): 스테이지 상태(진행/클리어) 관리
- `ClearConditionModule`(Module, 순수 로직): 생존 적 수 카운트, 0 도달 시 클리어 판정
- 적 사망 이벤트(Task 2/4) 구독 → 잔존 수 감소 → 전멸 시 클리어 이벤트 발행
- 클리어 이벤트는 보상 선택(Task 7) 트리거로 연결

**컨텐츠 의존성**
- **Task 2/4**: 적 사망 이벤트(전멸 카운트의 소스)
- **Task 4**: 스테이지에 배치된 적 집합

**검증 게이트**: 적 전멸 시 클리어 이벤트 발행

---

## Task 7 — 보상 선택 (2~3택)

| 항목 | 내용 |
|---|---|
| 순서 | 7 (클리어에 성장 보상 부여, 로그라이크 루프 완성) |
| 시스템 | 보상 선택 |
| 목표 | 클리어 시 2~3개 보상 후보를 제시하고 선택을 적용한다 |

**내용**
- `RewardManager`(Manager, VContainer 등록): 클리어 이벤트 소비 → 후보 생성 → 선택 UI 트리거 → 적용
- `RewardModule`(Module, 순수 로직): 후보 풀에서 2~3개 추출, 선택 결과 적용 규칙
- MVP 보상 예: 공격력↑, 이동속도↑, 최대 체력↑, 스킬 쿨다운↓ 중 2~3택
- 선택 결과가 Task 1(이동)/Task 3(콤보 피해)/Task 5(스킬)의 수치에 반영되도록 연결
- 선택 후 다음 스테이지 진입으로 연결(MVP는 1회 선택까지)

**컨텐츠 의존성**
- **Task 6**: 클리어 이벤트 (보상 트리거)
- **Task 1/3/5**: 보상이 반영될 대상 수치
- UI: 보상 선택 화면

**검증 게이트**: 클리어 → 2~3택 표시 → 선택 보상이 수치에 반영
